本章节主要介绍 Kotlin 的内置类型和简单用法

变量的声明

Kotlin 的变量声明方式，有点类似于 TypeScript，是比较现代的一种做法，一般形式为修饰符 变量名: 类型 = 值，其中，类型声明可以省略。

修饰符有两种

  • val：只读变量
  • var：可读写变量，定义时必须指定值，且不可更改

与 Java 对比

易混淆的 Long 类型标记

在 Java 里，Long 类型结束使用 l 是可以编译通过，只是不推荐（小写的 l，看起来就跟数字 1 一样）

在 Kotlin 里面，直接就编译不通过，强制要求修改为 L

Kotlin 数值类型转换

在 Java 里，int 类型可以隐式转换为 long

而到了 Kotlin 这，对不起，不支持

无符号类型（兼容 C Native）

从 v1.3 开始，Kotlin 支持无符号类型

集合框架

Kotlin 在 Java 集合的基础上，做出了一些增强，具体表现为以下几点

  • 增加了 “不可变” 集合框架的接口
  • 没有另起炉灶，复用 Java Api 的所有实现类型
  • 提供了丰富医用的方法，例如 forEach/map/flatMap 等

Scala 也是一门 JVM 语言，Kotlin 很多特性都参考了 Scala
